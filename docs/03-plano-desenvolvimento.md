## 4. Estratégia de Desenvolvimento

### Ambiente de desenvolvimento

VScode, controle de versão será através de 0.1, 0.2 (Versão final será 1.0), correção de bugs posteriores usaram o mesmo padrão.

### Estratégia de testes

Iremos testar em diferentes computadores (pessoais e da PUC). Testes para achar possíveis erros de lógica nos jogos. Além disso, iremos desenvolver o site por partes, evitando empilhar diferentes estruturas em um mesmo teste.

### Estratégia de deploy

Será hospedada no Github pages, por ser grátis e de fácil manipulação.
